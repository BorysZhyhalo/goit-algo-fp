# Домашнє завдання 6— Жадібні алгоритми та динамічне програмування

## Опис

Програма вирішує задачу вибору їжі з максимальною сумарною калорійністю в межах бюджету двома підходами:

1. **Жадібний алгоритм** — вибір за співвідношенням `calories / cost`
2. **Динамічне програмування** — оптимальний набір страв (задача про рюкзак 0/1)

## Файли

| Файл | Призначення |
|------|-------------|
| `main.py` | Функції `greedy_algorithm()` та `dynamic_programming()` |

## Запуск

```bash
cd food_selection
python main.py
```

Після запуску введіть бюджет, наприклад `100`.

## Дані

```python
items = {
    "pizza": {"cost": 50, "calories": 300},
    "hamburger": {"cost": 40, "calories": 250},
    "hot-dog": {"cost": 30, "calories": 200},
    "pepsi": {"cost": 10, "calories": 100},
    "cola": {"cost": 15, "calories": 220},
    "potato": {"cost": 25, "calories": 350},
}
```

## Алгоритми

### `greedy_algorithm()`

1. Сортує страви за спаданням `calories / cost`
2. Додає страви, поки сума вартості не перевищує бюджет

**Плюс:** швидко  
**Мінус:** не завжди дає оптимальний результат

### `dynamic_programming()`

1. Будує таблицю `dp[i][w]` — максимум калорій для перших `i` страв і бюджету `w`
2. Відновлює набір страв з таблиці

**Плюс:** оптимальне рішення  
**Мінус:** більше пам'яті та часу

## Приклад (бюджет 100)

| Алгоритм | Страви | Калорії | Вартість |
|----------|--------|---------|----------|
| Greedy | cola, potato, pepsi, hot-dog | 870 | 80 |
| DP | potato, cola, pepsi, pizza | **970** | 100 |

DP знаходить більше калорій, повністю використовуючи бюджет.

## Висновки

1. **Жадібний алгоритм** добре працює як швидке наближення, але може пропустити кращу комбінацію страв.
2. **Динамічне програмування** гарантує максимальну калорійність для заданого бюджету.
3. На бюджеті 100 greedy дає 870 калорій, а DP — 970, що показує різницю підходів.
4. Обидва алгоритми дотримуються обмеження `total_cost <= budget`.
